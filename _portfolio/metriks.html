Analytics service for tracking and reporting

Overview

  • HTML
  • CSS
  • Javascript
  • RESTful API
  • Ruby on Rails


A web tool that allows activity tracking on registered web applications. Designing and building an API from scratch posed some challenges. The biggest issue would be the management and communication of data between the server-side and the client-side.

The entire service was constructed using Ruby on Rails. In order for the service to track events from registered applications, custom API routes and a controller were added, along with a custom Javascript snippet that would be included on the client side. This Javascript snippet tracks an event which then makes an AJAX request to the server-side enabling the event to be added to the user's registered application. To display tracked events so that users can visualize the data, graphs were used. The Chartkick gem was installed that allowed the creation of graphs with the number of tracked events on each application.
